## 2. 产品定位

### 2.1 一句话定位

TeachZhao 是一个面向学习者和教育工作者的 AI 教师平台，通过智能对话、个性化讲解、课程生成、学习陪练与进度追踪，帮助用户获得随时可用、可互动、可复盘的学习体验。

### 2.2 产品价值

- 对学生：获得低门槛、即时响应、可反复追问的 AI 老师。
- 对家长：降低辅导压力，获得孩子学习过程和薄弱点反馈。
- 对教师：快速生成课程内容、练习题、讲解脚本和个性化辅导建议。
- 对机构：用 AI 教师能力提升课程交付效率，降低单个学员服务成本。

### 2.3 核心卖点

- AI 教师实时讲解：像真人老师一样解释概念、拆解题目、引导追问。
- 个性化学习路径：根据用户水平、目标和错误记录调整内容难度。
- 多学科适配：支持语言、数学、科学、编程、考试备考等学习场景。
- 互动式学习体验：对话、问答、测验、反馈、复习计划形成闭环。
- 教师工作台：支持课程生成、题目生成、讲义整理、学习报告输出。

## 3. 目标用户

### 3.1 学生用户

- K12 学生：需要作业辅导、知识点讲解、错题解析。
- 大学生：需要课程辅助、论文思路、编程学习、考试复习。
- 自学者：需要系统学习路径、实时答疑和练习反馈。

### 3.2 教师用户

- 独立教师：希望快速备课、生成练习、提升教学效率。
- 培训机构老师：希望标准化课程内容并提供课后 AI 辅导。
- 内容创作者：希望将课程内容转化为可互动的 AI 学习体验。

### 3.3 付费决策者

- 家长：关注学习效果、安全性、可控性、使用成本。
- 学校/机构负责人：关注规模化交付、数据看板、班级管理和品牌化。

## 6. 核心功能说明

### 6.1 AI 对话讲解

用户输入问题后，AI 教师以循序渐进方式回答。回答结构包括：概念解释、步骤拆解、常见误区、练习建议和追问入口。

预期交互：

- 输入框支持发送问题。
- 支持快捷问题标签。
- AI 回复显示打字动画。
- 支持“再讲简单一点”“给我例题”“生成测验”等快捷操作。

### 6.2 课程生成

教师或学生输入学习目标后，系统生成课程大纲、每节课目标、讲解内容、练习题和复习计划。

预期交互：

- 选择科目、年级、目标、难度。
- 点击生成后展示课程卡片。
- 支持展开课程详情。

### 6.3 智能练习与测验

系统根据学习主题生成选择题、填空题、问答题或编程题，并根据用户答案给出反馈。

预期交互：

- 答题后显示正确/错误状态。
- 显示解析和知识点归属。
- 错题进入复习列表。

### 6.4 学习进度追踪

展示用户学习时长、完成课程数、掌握度、薄弱知识点和推荐复习内容。

预期交互：

- 图表展示掌握度。
- 卡片展示下一步建议。
- 支持按科目筛选。

### 6.5 教师工作台

面向教师提供备课、出题、讲义生成、班级学习报告等能力。

预期交互：

- 创建课程。
- 生成讲义。
- 生成课堂测验。
- 查看班级学习情况。

## 7. 视觉设计方向

### 7.1 整体风格

现代 AI SaaS 风格，强调清晰、可信、轻盈和教育亲和力。界面应避免过度花哨，保持学习产品的专业感，同时通过柔和插画、微动效和高质量卡片增强亲和力。

### 1. 角色设定
你是一位专注于 **Claymorphism（黏土形态）** 风格的 UI/UX 设计专家，擅长打破传统扁平化设计的枯燥，通过模拟物理世界的 **黏土质感、蓬松形态和柔和光影**，创造出极具亲和力和趣味性的数字界面。你的设计哲学是「**数字世界的实体化与软化**」，旨在让用户感受到屏幕背后的元素仿佛是真实存在的、可触摸的柔软物体，而非冰冷的像素。

### 2. 场景定位
**适用场景**：
- **Web3、NFT 与元宇宙项目**：利用其独特的 3D 属性和新潮感，展示虚拟资产或创建沉浸式体验。
- **儿童教育与早教应用**：柔软、圆润的视觉语言天然具有安全性与亲和力，能降低儿童的认知负担。
- **创意作品集与个人博客**：通过强烈的视觉风格展示设计师的个性和前卫审美。
- **休闲游戏界面**：与游戏内的卡通渲染风格高度契合，增强整体的娱乐氛围。
- **健康与冥想应用**：柔和的色彩和圆滑的形态有助于传递放松、舒缓的情绪。

**不适用场景**：
- **数据密集型后台管理系统（Dashboard）**：厚重的阴影和边距会占用大量屏幕空间，降低信息密度。
- **严肃的金融或法律文档处理**：过于活泼和童趣的风格会削弱权威感与专业严肃性。

### 3. 视觉设计理念
Claymorphism 的核心在于**「蓬松感」**与**「悬浮感」**。
- **反扁平化**：拒绝锐利的边缘和完全平坦的色块。每个 UI 元素都应该像是一个充气的气球或一块捏好的橡皮泥。
- **极致圆润**：所有的矩形都应该拥有巨大的圆角（Border Radius），按钮甚至可以接近胶囊形或圆形，模拟手工捏制的自然弧度。
- **空间层次**：通过强烈的深度暗示，让元素看起来是「漂浮」在背景之上的，或者是从背景中「凸起」的实体。

### 4. 材质与质感
要实现逼真的黏土质感，必须精细控制光影（CSS `box-shadow` 的高级应用）：
- **哑光表面（Matte Finish）**：黏土不是塑料或玻璃，不应有强烈的高光反射。表面应呈现漫反射，质感细腻且略带粗糙度。
- **双重阴影机制**：
  - **外部阴影（Drop Shadow）**：使用深色、大模糊半径的投影，营造物体悬浮的高度感。通常使用两个投影：一个深色模拟遮挡，一个浅色模拟环境光反射。
  - **内部阴影（Inner Shadow）**：这是 Claymorphism 的灵魂。在元素内部上方添加浅色内阴影（高光），在内部下方添加深色内阴影。这能营造出物体边缘的**厚度**和**圆滑的倒角**，产生类似 3D 建模的充气效果。
- **色彩策略**：偏向使用**高明度、低饱和度**的马卡龙色系（Macaron Colors）或粉彩（Pastel Colors）。背景色通常与主体色系保持一致但略浅，以减少视觉冲突，增强整体的柔和感。

### 5. 交互体验
交互设计应模拟物理按压的反馈：
- **Hover（悬停）**：元素轻微上浮，阴影扩散，暗示它是可交互的实体。
- **Active（点击/按压）**：这是体验的关键。点击时，元素不应只是变色，而应模拟**被按下去**的物理状态。
  - **视觉变化**：外部阴影收缩或消失，内部阴影加深，模拟物体被压入平面或变扁的状态。
  - **动态效果**：使用弹性曲线（Spring Animation），让按钮在松开时有「回弹」的果冻感。
- **表单输入**：输入框看起来应该像是黏土表面被「挖」出的一个凹槽（Neumorphism 的凹陷风格的柔软变体）。

### 6. 整体氛围
Claymorphism 营造的是一种**温暖、乐观、安全且充满好奇心**的氛围。它唤起了用户童年玩橡皮泥的触觉记忆，消除了技术的冰冷距离感。整个界面应该像是一个精心布置的玩具屋，每一个组件都让人忍不住想去戳一下、捏一下。这种风格传递出的情感是：「别紧张，这里很有趣，随便玩。」



## 8. 技术实现方案

### 8.1 推荐技术栈

建议采用 React + Vite 开发，便于后续扩展 AI Demo、登录、价格页和多页面体验。

- 前端框架：React + Vite。
- 样式方案：Tailwind CSS 或 CSS Modules。
- 图标：lucide-react。
- 动画：Framer Motion 或 CSS transition。
- 路由：React Router。
- 状态管理：组件内状态优先，复杂后再引入 Zustand。
- 构建部署：Vite 静态构建，可部署到 Vercel、Netlify 或自有服务器。

如果只需静态复刻，也可以使用 HTML + CSS + JavaScript，但扩展性低于 React 方案。

### 8.2 页面组件拆分

- `Header`：顶部导航、移动端菜单。
- `HeroSection`：首屏文案和产品视觉。
- `DemoPanel`：AI 对话模拟、课程卡片、学习数据。
- `FeatureGrid`：核心功能卡片。
- `UseCases`：目标用户场景。
- `WorkflowSection`：学习流程。
- `Testimonials`：用户评价。
- `PricingSection`：价格套餐。
- `FAQSection`：可展开问答。
- `Footer`：页脚链接。

### 8.3 交互实现清单

- 顶部导航锚点跳转。
- 移动端汉堡菜单展开/收起。
- CTA 按钮点击状态。
- Demo 聊天框输入和模拟回复。
- 功能卡片悬停反馈。
- 场景 Tabs 切换。
- 价格月付/年付切换。
- FAQ 手风琴展开。
- 页面滚动淡入动画。
- 表单基础校验。

### 8.4 响应式断点

- 移动端：小于 640px，单列布局，导航收进菜单。
- 平板：640px-1024px，双列或紧凑网格。
- PC：大于 1024px，多列布局，完整导航和大尺寸产品演示。

### 17.4 核心功能使用

1. **首页浏览**：打开首页可看到完整的品牌介绍、角色展示、产品演示、核心功能、用户评价、价格方案和 FAQ。
2. **进入角色系统**：点击首页「进入角色系统」按钮或导航栏链接，进入角色列表页面。左侧是角色列表（支持搜索），右侧是角色详情（含认知网络和管理选项）。
3. **与角色对话**：在角色详情页点击「进入对话」，或在导航中点击聊天图标。可输入消息与 AI 角色对话，支持快捷提问、新建话题。
4. **课程生成**：首页演示区可输入学习目标快速生成课程大纲。
5. **登录/注册**：点击导航栏「登录」按钮，输入邮箱即可登录（首次自动注册），支持身份选择。
6. **价格切换**：价格区支持月付/年付切换，实时显示不同价格。
7. **充值**：登录后进入账户页，点击「+ 充值」按钮可选择金额充值（演示模式）。
8. **FAQ**：首页底部和价格页包含常见问题，点击可展开/收起。
9. **移动端**：缩小浏览器窗口或在手机上访问，导航会自动切换为汉堡菜单，布局自动适配。
